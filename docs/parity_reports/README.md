## Report Naming Convention

Reports follow the naming pattern: `report_YYYY-MM-DD.md`

Example: `report_2026-02-27.md`

---

## Report Contents

Each report includes:

1. **Summary Statistics**
   - Total events on each platform
   - Parity score percentage
   - Number of mismatches

2. **Detailed Findings**
   - Android-only events (if any)
   - iOS-only events (if any)
   - Schema mismatches (if any)

3. **Matched Events Table**
   - Complete list of all events present on both platforms
   - Mapping of constant names (Android uses SCREAMING_SNAKE_CASE, iOS uses camelCase)

## How to Generate Reports

### Automated (Weekly)

Reports are automatically generated every Monday at 9:00 AM UTC via GitHub Actions.

### Manual

```bash
# Generate console report
python scripts/audit_event_parity.py

# Generate markdown report
python scripts/audit_event_parity.py --format markdown --output docs/parity_reports/report_$(date +%Y-%m-%d).md

# Generate JSON report
python scripts/audit_event_parity.py --format json --output docs/parity_reports/report_$(date +%Y-%m-%d).json
```
